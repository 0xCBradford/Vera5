# Acceptance checklists (internal)

This directory holds **internal** week sign-off checklists for Vera5 development. These files are not public user documentation—do not link them from `README.md`, the extension, or store listings.

## Naming

| Pattern | Use |
|---------|-----|
| `W0_7_1.md` | Week 0, Day 7 checkpoint (first sign-off artifact) |
| `W{n}_7_1.md` | Week *n* Day 7 checkpoint |

Add checklist files when each week’s Day 7 task runs. Each checklist should record verification evidence (commands, paths, pass/fail) and sign-off status.

## Hygiene

Checklists may reference internal build identifiers for traceability. Keep product-facing docs free of that scaffolding (see repository `SECURITY.md` and root `README.md` for public policy).
